# understand_report/前提.md

## 前提
実装忘れたということで、以下の理解が部分的・断片的な状態にある。

| レイヤー | 理解対象 | 具体例 |
|---|---|---|
| 文法 | コードを読めること | async/await, class, interface |
| 構造 | ファイル・責務構造 | frontend/backend/domain |
| データ流れ | 入出力と変換 | API → Service → DB |
| 実行開始点 | どこから処理が始まるか | main(), index.tsx |
| 状態管理 | 状態の保持と更新 | Redux, ViewModel |
| 依存 | 呼び出し関係 | DI, Repository |
| 編集地点 | 修正影響範囲 | controller/service |

現在は、
「動かせるが説明できない」
「修正できるが理由を言語化できない」
というスポンジ化状態にある。

---

## 今回の目的

このプロジェクトのコード理解を通じて：

- 実装の流れを説明できる
- なぜその設計になっているか説明できる
- 修正時の影響範囲を判断できる
- 同様の設計を別プロジェクトでも再利用できる

状態になることを目的とする。

---

## 理解完了条件

以下を自力で説明できること：

- 起動から画面表示までの流れ
- 1機能のデータフロー
- 状態がどこで変更されるか
- API/DBとの接続地点
- なぜその責務分離になっているか
- どこを変更すれば機能追加できるか

---

## 成果物

理解内容は以下として記録する：

- 実行フロー図
- ディレクトリ責務一覧
- 状態管理一覧
- API呼び出し一覧
- 依存関係メモ
- 機能別の処理追跡ログ

これらを `understand_report/` 以下に保存する。
