# `docs/next_connection/` — 引き継ぎ資料

セッション間で「未解決の問題」「次にやるべきこと」「ハマりどころ」を
確実に引き継ぐためのディレクトリ。新しい Claude セッション・新しい開発者が
作業に入る前に必ずここを確認する運用にする。

---

## 使い方

### 作業を始める前に
1. `PROMPT.md` を開いて手順を確認
2. `目次.md` で今回触る領域に関連するキーワードを検索（ファイル名・機能名・エラーメッセージ等）
3. ヒットした `index/active/` 配下のファイルを精読

### 作業中に未解決問題・回避策が出たら
作業終了時に必ずここへ記録する。次のセッションが同じ罠を踏まないため。

---

## ディレクトリ構成

```
docs/next_connection/
├── README.md          このファイル
├── 目次.md            全エントリの索引（最初に見る）
├── PROMPT.md          作業開始時の参照プロンプト
└── index/
    ├── active/        作業中・未解決
    │   ├── llm/
    │   ├── bug/
    │   ├── ui/
    │   ├── infra/
    │   └── analysis/
    ├── resolved/      解決済みだが参考価値あり
    └── archive/       古い・参照頻度低
        ├── bug/
        └── analysis/
```

### カテゴリの目安

| カテゴリ | 対象 |
|---|---|
| `llm` | LM Studio・Anthropic API・プロンプト・LLM設定 |
| `bug` | バグ調査・原因分析・修正記録 |
| `ui` | フロントエンド・コンポーネント・スタイル |
| `infra` | Cloudflare・Supabase・wrangler・デプロイ・運用 |
| `analysis` | パターン検出・特性推定・分析ジョブ |
| `misc` | 上記に当てはまらないもの |

### ステータス遷移

```
active  ──解決──▶ resolved ──参照頻度低下──▶ archive
   │
   └──作業継続のまま記録更新
```

- **active**: 未解決問題が残っている、または現在進行中の作業
- **resolved**: 解決済みだが「なぜそうなったか」「どう直したか」が将来参考になる
- **archive**: 古くて今後参照する可能性が低いが、履歴として残す

## 新規追加手順

1. `index/active/<category>/YYYY-MM-DD_<kebab-slug>.md` を作成
2. ファイル先頭にフロントマターを記入（下記テンプレ）
3. `目次.md` の該当カテゴリ行に1行追加

### フロントマターテンプレ

```yaml
---
title: わかりやすい日本語タイトル
category: llm            # llm | bug | ui | infra | analysis | misc
status: active           # active | resolved | archived
date: 2026-06-14         # 最終更新日 (YYYY-MM-DD)
tags: [keyword1, keyword2]
related: []              # 関連ファイル相対パス
---
```

### ファイル本文の推奨セクション

```markdown
## 背景・目的
何をしようとしてこの作業が始まったか。

## 今回セッションで実施した内容
具体的な変更点・コマンド・設定。

## 現在の状態
- 動いていること ✓
- 動いていない・未確認 ✗

## 未解決の問題と検証優先度
🔴 HIGH / 🟡 MEDIUM / 🟢 LOW で優先度を明示。
各問題に「症状」「仮説」「検証方法」を書く。

## 次のセッションで最初にやること
箇条書きで1〜5項目。
```

---

## ステータス変更時

`active` → `resolved` / `archive` に変えるときは:
1. フロントマターの `status` を更新
2. ファイルを `index/active/<cat>/` から `index/resolved/` または `index/archive/<cat>/` に `git mv`
3. `目次.md` のステータス列を更新（または該当セクションへ移動）

---

## 永続ドキュメントとの分離ルール

このディレクトリには **「引き継ぎ性のあるもの」だけ** 置く。
それ以外（永続ドキュメント）は `docs/` 直下のまま:

- ビジネス分析・リーンキャンバス
- アーキテクチャ図 (`*.mmd`)
- 評価データ (`eval/`)
- 永続的な仕様書

判断基準: **「これは将来書き換えられて陳腐化するか？ それとも継続的に参照されるか？」**
陳腐化する性質のものは next_connection に。
