### issues (直面している問題)
- チャット画面でメッセージを送信しても、AIからの回答ではなく、常に**固定のテンプレート文**（「仮説を作るために、次の点を記録してください…」）しか返ってこない。
- または、内部的に `{"message":"予期しないエラーが発生しました"}` が発生し、チャットが正常に進行しない。
- UIから設定したLM StudioのAPIキーが正しく保存・適用されていない。

### target (問題が起きている箇所・環境)
- **環境:** Cloudflare Workers プレビュー環境 (`npm run preview`)
- **機能:** LLM設定の読み込み処理、およびチャット送受信機能
- **関連コード:** 
  - `src/infrastructure/llm/resolveStoredLlmConfig.ts` (DB設定の読み出し)
  - `ChatUseCase` (チャット実行とエラーハンドリング)
  - `app/api/settings/llm/route.ts` (設定保存)

### 仮説 (原因のメカニズム ※検証・修正済み)
1. **APIキー不足によるエラー発生:**
   LM StudioのようなローカルLLMは本来APIキーを必要としない（またはダミーでよい）が、DB上の設定レコードにAPIキーが存在しない (`hasApiKey=false`) 状態になっていた。そのため、設定を読み込む際にシステムが「APIキーがない」と判断し、例外（エラー）を投げていた。
2. **フォールバックの暴発:**
   チャット処理のコアである `ChatUseCase` が上記の例外をキャッチすると、「LLMと通信できない状態」だと判断し、フェイルセーフとして**フォールバックモード（あらかじめ用意された固定のテンプレートを返すだけのモード）**に切り替わっていた。
3. **結果:**
   LM Studioには一切リクエストが飛んでおらず、何度チャットを送ってもシステムが自律的に「ずっと同じテンプレートを返す」状態に陥っていた。

---
**現在のステータス:**
この仮説をもとに、先ほど「APIキーがなくても例外を投げずに設定を通す」ようにコードを3箇所修正しました。ターミナルを一度落として `npm run preview` で再ビルド・再起動すれば、フォールバックモードを抜けてLM Studioから本当の回答が返ってくるようになります。
